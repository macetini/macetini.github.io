Overview

DarkOrbit was the most challenging project I worked on during my time at Bigpoint Games. It required a deep understanding of both back end and front end development. I was responsible for implementing new features, fixing bugs, and optimizing the game's performance. I also worked closely with the design team to ensure that the game met the client's requirements and provided a satisfying user experience.

Responsibilities

It was my first role as a Game Developer, and not just sole front end. While the bulk of my work was on the front end, I also contributed to the back end. Mainly in the form of API development and database management. This was the most complex project I worked on at Bigpoint, and it taught me a lot about full stack development. There have been times when I would arrive at work a 8 in the morning and leave at 10 at night. But it was worth it, as I was able to learn a lot and contribute to a successful project.

  • Modification and improvement of client ActionScript3 code with Apache Flex framework.
  • Development of PHP Website and Java Server side.
  • Implementation of new PHP configs.
  • Modification of HTML/CSS/JS layer.
  • Modification of MySQL database data .
  • Addition of multiple smaller flash features.
  • Simple Image manipulation with Photoshop and Gimp.
  • Implementation/modification of new flash content by using Adobe Flash CC.
  • Analyzing and optimizing the AS3 code with Adobe Scout.
  • Using the Jira workflow - accepting/creating tickets.
  • Communicating with the artists on the new content for the games.
  • Creation/Reintegration of new branches with Git.
  • Syncing release versions to live/social/stage.

Big UI Redo

One of the biggest features I worked on was the complete redesign of the game's user interface. This was a massive undertaking that required a lot of coordination with the design team. I was responsible for implementing the new UI in Bigpoint internal custom framework, ensuring that it was both functional and visually appealing.

The new logic included modifications on both back end and front end. This made it easier to manage and maintain. The new UI was a huge success, and it helped to improve the overall user experience.

Additionally a new crafting system was added.

Multi-Item Sell System

The multi-item sell system was another significant feature I worked on. This system allowed players to sell multiple items at once and improving the overall user experience.

The system required a lot of work on both the back end and front end. We had user that accidentally bought 5000 items which would crash the client, as it was not intended for such use cases.

Other Features

A lot of other features were added to the game. Too many to list here. One of the notable ones was the jump system, which allowed players to jump from one system/server to another. It was a complex feature that required significant changes.

Server Merge

DarkOrbit conducted a series of server merges primarily in 2018. The merges were announced and took place in waves. All programmers in department were involved and did little else. Most of the work was done on the backend and the databases.

The server merges were complex and required careful planning and execution to ensure a smooth transition for players. The merges involved combining player data, game progress, and in-game assets from multiple servers into a single server. This process required extensive testing and coordination with various teams within Bigpoint to minimize downtime and ensure data integrity.
