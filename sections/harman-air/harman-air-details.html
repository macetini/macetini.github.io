Overview

Back in the day (2014), I created a few games using Adobe AIR. I wanted to aquatinted myself with a new stage 3d platform. Today Adobe AIR has been taken over by Harman, and I decided to revisit one of my old projects and bring it back to life using the new Harman AIR SDK.

PONG

This is a small PongActionScript 3 MVCS game (Pong) using:

  • Starling
  • Feathers for rendering/UI
  • Robotlegs
  • Signals
  • Swiftsuspenders for DI and app structure

Originally created as an experiment in 2014. The project demonstrates the use of the Robotlegs MVC pattern alongside Stage3D and Signals for a clean, modular architecture. The project dependency code had to be modified to work, so '.swc' files were not used. They are still included in the repo for reference.

While visually not that impressive, there is a lot under the hood.

Features & Tech Stack

  • Harman Adobe AIR (v51.2.1.0)
  • Starling Framework
  • Feathers UI
  • Robotlegs (MVCS)
  • SwiftSuspenders (Dependency Injection)

Entry / boot sequence:

  • Main creates Starling and Robotlegs context and runs configuration in org.pong.mvcs.AppConfig.
  • App config fragments DI/bootstrapping into small bootstrap classes under bootstraps:
  • Models: org.pong.mvcs.controller.bootstraps.BootstrapModels
  • Signals: org.pong.mvcs.controller.bootstraps.BootstrapSignals
  • Signalâ†’Command bindings: org.pong.mvcs.controller.bootstraps.BootstrapSignalCommandMap
  • Mediator mappings: org.pong.mvcs.controller.bootstraps.BootstrapMediatorMap
  • Services: org.pong.mvcs.controller.bootstraps.BootstrapServices
  • Utils: org.pong.mvcs.controller.bootstraps.BootstrapUtils

The system can be adapted for any kind of project. It was created with modularity in mind, allowing for easy extension and customization.
