Overview

One of my first tasks as a Game Developer was development and optimization of Flash client games. The first game I worked on was Skyrama, an online browser game from Bigpoint Games.

Skyrama is an online browser game made with 90s inspired Art Style.

To the smaller degree I was involved in both client and server-side development, but my main focus was on client-side performance optimization

Responsibilities

  • Modification and improvement of client ActionScript3 code with Apache Flex framework.
  • Debugging and analyzing PHP server side code.
  • Implementation of PHP configs.
  • From time to time implementation of new PHP server code features.
  • Modification of HTML/CSS/JS layer.
  • Modification of MySQL database data and creation of new Tables.
  • Implementation of client/server tracking log system.
  • Addition of multiple smaller flash features.
  • Simple Image manipulation with Photoshop and Gimp.
  • Implementation/modification of new flash content by using Adobe Flash CC.
  • Analyzing and optimizing the AS3 code with Adobe Scout.
  • Using the Jira workflow - accepting/creating tickets.
  • Communicating with the artists on the new content for the games.
  • Creation/Reintegration of new branches with both Git and SVN.
  • Syncing release versions to live/social/stage.

Large Planes feature

The Large Planes feature was a significant addition to the game, allowing players to explore vast, open skies with detailed aircraft models. I spend good part of my time optimizing the performance of this feature to ensure smooth gameplay even with the increased graphical demands.

This feature not only enhanced the visual appeal of the game but also added new gameplay dynamics, as players could now manage larger aircraft with unique handling characteristics.

New Logging System

To improve debugging and performance monitoring, I developed a new logging system that tracked client-side events and errors. This system provided valuable insights into user behavior and helped identify performance bottlenecks.

The logging system was designed to be lightweight and efficient, minimizing its impact on game performance while still capturing essential data. It required modifications on both client and server side.
